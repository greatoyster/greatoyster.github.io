Rust 错误处理

在这之前有一个必要重要的点需要交代一下

这个enum表示结果要么是对的要么是错的，T和E是类型变量（模板变量），如果结果符合要求那么就会用到关联的类型T, 反之就是Err(E)

那么当进行一个操作之后，我们就有必要对他的结果进行探讨（对了的话程序继续，错的的话返回错误）

这段代码就是对打开文件这一操作做了一个分类讨论，对了的话就打开文件，错了的话就panic。

这里面panic!会让程序中断，可以在运行的时候打开回溯开关来检查此时的堆栈信息

?操作符对一个合法的值进行unwrap，或者当是错误类型的时候，向上传播错误，该操作符仅能作用于类型Result<T, E>和Option<T>

当作用于Result<T, E>类型的值时：

  • 如果值是Err(e)，则该函数或者闭包返回 Err(From::from(e))
  • 如果值是Ok(x)，则将值unwrap为x

当作用于Option<T>类型的值时：

  • 如果值是None，则该函数或者闭包返回None
  • 如果值是Some(x)，则将值unwrap为x

特别的是：

  • 同一个函数中使用?操作符的地方必须返回同一个错误类型，否则会编译失败
  • main函数有两个有效返回值，分别为()，Result<T, E>
