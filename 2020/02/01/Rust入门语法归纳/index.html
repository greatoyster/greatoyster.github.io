Rust入门语法归纳

前言

Rust是一门学习曲线极其陡峭的编程语言，里面的语法规则比较庞杂。不得不承认，Rust可读性是很高的，但是写出正确的代码又是另外一回事了。

好在学习Rust有比较好的官方教材《The Rust Programing Language》，而且也有了翻译质量很高的中文版，但是我发现这本官方教材里面很多东西都是零零碎碎的，例子很多，却缺乏了一些很general的表述。在这里，我负责来汇总一下Rust的部分基本语法规则。

此外，如果在一门或者多门其他编程语言上已经有了相当的基础，那么学习Rust会快上一些。这也是本文面向的主要对象。

友情提示：把Rust作为一门入门语言是一个不太理智的决定。

类型系统

类型签名

遵循变量名: 变量类型的格式，冒号前面是变量名，后面是变量类型

在C语言中的char a到了Rust里面就变成了a : char

这与Haskell里面的类型签名是一致的。

内置类型（built-in type）

也叫做原生类型（primitive）。这些类型包括了内建在Rust语言中的和标准库里提供的两大类。

布尔类型

bool: true or false

字符

char:代表一个Unicode字符，与其他编程语言不同，它的大小是4字节（意味着可以代表更多的字符，比如表情）。

数字

数字类型名的组成规则是 数字类型+有效位数

数字类型有三种：u, i, f, 分别代表无符号整数，正数，浮点数

数字类型的几个例子：u8, f64, i32, 诸如此类，用C语言的思路来看的话，这三个类型就是unsigned char, double, int

Rust数字类型的命名规则是直观而易于理解的。

数组

数组默认不可修改，他有两种属性：元素类型和长度，也就是 [元素类型;长度]

比如一个数组[1，2，3]，它的类型签名就是[i32;3]

如果要用动态数组，就需要用到vec!的宏(或者Vec<T>)，直接用例子来演示

函数类型

一个函数类型可以表示为fn (参数类型)->(返回值类型)

对应C语言中的函数指针。

字符串类型

Rust里面有两种主要的字符串类型：String和str，其中str是内建类型，而String由标准库提供。

其中String类型是分配在堆(heap)上的，它是可变的，还有所有权，而str一般以被借用的形式出现，即&str。
